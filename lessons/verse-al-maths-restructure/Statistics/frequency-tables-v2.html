Driving Question: How can a table turn messy data into a clear story?

üß† The Knowing World

Focus Concept (KS4 Statistics): Using frequency tables (including relative and cumulative frequency) to organise and describe data.

Seed of Change (KS4): Interpret and construct tables and other statistical diagrams; use measures and representations to describe distributions.

Key language:

  • Raw data – the original list of values (e.g. 3, 5, 5, 7, 8…)
  • Frequency – how many times a value appears in the data.
  • Relative frequency – frequency as a fraction or percentage of the total.
  • Cumulative frequency – running total of frequencies up to a certain value.

Using the Frequency Table Builder:

  1. Type a small data set into the app (e.g. “3, 5, 5, 7, 8, 8, 8, 10”).
  2. Click to build the frequency table.
  3. Notice how the raw list becomes a table of values and frequencies.
  4. Explore the relative and cumulative frequency columns if they are shown.

üí¨ Prompt: What patterns are easier to see in the table than in the raw list of numbers? How does the table help you “see” the data more clearly?

üåç The Human Story

Frequency tables are used in many real settings:

  • How often buses arrive early, on time or late.
  • How many patients wait different lengths of time in a clinic.
  • How many students get each grade in an exam.
  • How many customers buy 0, 1, 2, 3 items in a shop.

Instead of just saying, “Some people waited a long time,” a frequency table can show exactly how many and how often.

üó£ The Expressive Self

üé® Imagine your day as data. Choose something you do repeatedly, like:

  • Number of messages you send each hour.
  • Number of steps you walk in different parts of the day.
  • Number of songs you listen to each evening.

Now:

  1. Sketch a tiny “raw data diary” of the numbers across a day.
  2. Turn it into a simple frequency table on paper:
    • Values (e.g. “0 messages, 1–5 messages, 6–10 messages”)
    • Frequencies (how many times each happened)
  3. Decorate or annotate it with colour or emojis to show how the data feels (busy, chilled, spiky, flat).

üí¨ Prompt: Does your table make your day look calmer or more intense than it feels from the inside?

üèó The Designed World

Now we’ll use frequency tables as design tools to model and compare real situations.

Step 1 – Grouped data with class intervals (Activity 2)

Copy of Grouped Frequency Table Interactive Web App for GCSE Maths by Gerry Docherty

Using the Grouped Frequency Explorer:

  1. Choose a data set (e.g. heights, times, or test scores) or enter your own.
  2. Choose a class width (e.g. 10 marks, 5 cm, 2 seconds).
  3. Generate the grouped frequency table with class intervals.
  4. Observe how the bar chart / histogram-style display changes when you choose:
    • Very narrow intervals (lots of classes, more detail).
    • Very wide intervals (fewer classes, less detail).

üí° Key idea: Grouped frequency tables are useful when there is a lot of data or continuous data. The choice of class intervals can change how the distribution looks, but the total frequency stays the same.

Step 2 – Comparing two groups (Activity 3)

Copy of Comparing Frequency Tables Web App for GCSE Maths by Gerry Docherty

Using the Comparing Frequency Tables app:

  1. Choose or enter data for two groups (e.g. Class A and Class B test scores, or two teams’ goals per match).
  2. Build a frequency table for each group.
  3. Look at the bar charts:
    • For which values does Group A have higher frequency?
    • For which values does Group B have higher frequency?
    • Which group seems to have more “high” values?

üë• Buddy Challenge: With a partner, choose one scenario in the app. Each of you writes a short summary answering: “Which group appears to perform better overall, and how do you know from the tables and charts?”

  • Stressed: “This looks like an exam question.”
  • Defensive: “I don’t like being compared.”

üõ† Final Task: From Raw Data to Comparison

Your Task: Use frequency tables to tell the story of one group, and then compare two groups.

Choose one pathway:

  1. Pathway A – One Group in Detail
    • Create or collect at least 20 pieces of data (real or fictional) on a topic you care about (e.g. daily screen time, number of steps, number of messages per day).
    • Use the Frequency Table Builder to turn your raw data into a frequency table.
    • Write a short explanation (6–8 sentences) describing what the table shows about your data: most common values, any surprising frequencies, overall pattern.
  2. Pathway B – Two Groups Compared
    • Invent or use two small data sets (at least 12 values each) that could represent two groups (e.g. Class A vs Class B scores; Team 1 vs Team 2 goals).
    • Use the Comparing Frequency Tables app to build tables and charts.
    • Write a comparison paragraph that answers:
      • Which group appears to perform better overall?
      • Which group has more high values?
      • Which group is more consistent or more varied?

What to Submit (Canvas Assignment):

  • The raw data you used (lists of values).
  • Your completed frequency table(s) (and grouped table if used).
  • Your written description or comparison paragraph.

Assessed using the GCSE Maths Rubric:

  • Conceptual Understanding: Do you correctly use frequency tables (and grouped tables if used) to describe data?

  • Application to Problems: Do you make sensible statements or comparisons based on the tables and charts?
  • Independence & Reflection: Do you reflect in your own words on what the data suggests, not just copy numbers?

We will explore how to describe, combine, and reason about sets—grouping ideas and membership logic.

Key ideas

  • Set: a collection of distinct objects (elements) often written using braces, e.g. {2, 4, 6}.
  • Element / Member: an item in a set, written “∈” (e.g. 4 ∈ {2,4,6}).
  • Empty set / null set: the set with no elements, denoted ∅ or {}.
  • Subset: A ⊆ B means every element of A is also in B.
  • Strict subset: A ⊂ B means A ⊆ B but A ≠ B (i.e., B has extra elements).
  • Union: A ∪ B = {x | x ∈ A or x ∈ B} (all elements in at least one set).
  • Intersection: A ∩ B = {x | x ∈ A and x ∈ B} (elements in both sets).
  • Complement / Universal set: If U is the universal set, then A′ or A‚Åª is {x ∈ U | x ∉ A}.
  • Venn diagrams: visual way to show union/intersection relationships.

üåç The Human Story

Sets help us organise and compare categories: who voted for which party, which students are in which club, or which products are eco-certified. In data justice, we often compare groups (sets): “Which demographics are included in a policy’s beneficiary set?”

Justice prompt: Suppose a city council offers free public transport to “residents earning < £20,000” (Set A) and “students” (Set B). Is someone who is both a low-income worker and a student counted twice or once? How can set logic help us ensure fairness in policy design?
